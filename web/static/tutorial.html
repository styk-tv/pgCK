CKP v3.8 — Interactive Intro

A concept kernel is a governed graph. This page talks to the live kernel at pgck.localhost over NATS WSS — every action below is a real seal, validated by SHACL, proven by HMAC. No REST.

0 · The wire

The browser publishes input.kernel.pgCK.action.<verb> and listens on result.kernel.pgCK.action.<verb>. A dev dispatcher maps verbs to ckp.seal / ckp.verify (the Rust CKA-4 dispatcher will replace it). Connection:

2 · The rule (a SHACL shape that cannot drift)

The board kernel's TaskShape requires part_of_goal + target_kernel (and the kernel requires task_id, priority, queue_seq, created_at). The gate runs inside the seal transaction — violations roll the whole write back.

3 · Add a valid instance → get a proof

Seal a complete Task. The kernel validates it, projects its link-triples, appends the HMAC ledger, and returns a proof digest.

4 · Break the rule → governed rejection

Same Task, but with part_of_goal removed. The SHACL gate rejects it — nothing is written. This is the point: malformed state cannot land.
