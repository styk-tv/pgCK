# v0.1.2 Release Issues

Release work is paused on the following blocker issues.

## Issue 1 — Proof Surface Must Match Reality

Status: completed

- Problem: the shipped proof method claims `ed25519+sha256`, but the current implementation uses HMAC and `ckp.verify()` does not verify the proof record or signature semantics.
- Acceptance:
  - The runtime ontology, SQL implementation, and tests agree on the shipped proof method.
  - `ckp.verify()` checks the latest durable proof/ledger state consistently.
  - Demo/runtime defaults stop deriving the identity key from `md5(project)`.

## Issue 2 — `ckp.validate()` Must Be Concurrency-Safe

Status: completed

- Problem: scratch validation graph ids are drawn from a small random pool, so concurrent validation calls can collide.
- Acceptance:
  - Concurrent backends do not share a scratch graph id.
  - The governed path still passes the existing SQL smoke gates.

## Issue 3 — Embedded NATS Must Enforce `max_payload`

Status: completed

- Problem: the embedded server advertises a 1 MiB max payload but accepts arbitrary client-declared payload lengths.
- Acceptance:
  - Oversized `PUB` frames are rejected before allocation.
  - The server tests cover the limit behavior.

## Issue 4 — Demo And README Defaults Must Match The Shipped Runtime

Status: completed

- Problem: the WSS companion docs still describe `:4222` while the shipped compose/runtime uses `:4223`, and the demo runtime ignores the documented `POSTGRES_PORT` fallback.
- Acceptance:
  - README, runtime defaults, and smoke commands describe the same ports.
  - The demo runtime honors the documented Postgres port override.

## Release Gate

Do not tag `v0.1.2` until all four issues are closed and the full verification set passes again.
